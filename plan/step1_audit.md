# 1단계 실행 결과: 현행 기능과 데이터 의존성 고정

## 실행 범위

현재 저장소에는 아래 파일만 존재한다.

- `data/vocab.json`
- `plan/implementation_plan.md`

따라서 기존 KamiVoca 앱 소스의 실제 의존성은 아직 코드 레벨로 확인할 수 없다. 이번 단계에서는 `plan/implementation_plan.md`에 명시된 기존 기능과 `data/vocab.json`의 실제 스키마를 기준으로 중국어 마이그레이션 기준을 고정한다.

## 유지할 기능 체크리스트

앱 소스가 추가되면 아래 기능을 회귀 기준으로 삼는다.

- 단원 기반 학습을 Step 기반 학습으로 유지한다.
- `lessonId` 1-20은 2개씩 묶어 Step 1-10으로 노출한다.
- 객관식 뜻 퀴즈를 유지한다.
- 복습 퀴즈와 복습 큐 저장을 유지한다.
- 학습 진행도, 점수, 랭크, 토스트, 리더보드를 유지한다.
- 유저 프로필을 유지한다.
- Firebase/Firestore 동기화를 유지한다.
- 오프라인 모드와 서비스 워커를 유지한다.
- 음성 재생과 사전 예열을 유지한다.
- 날씨 배경을 유지한다.
- 관리자 EDIT 탭은 중국어 필드 편집 기준으로 유지한다.
- 전역 삭제와 관리자 데이터 동기화를 유지한다.
- 한국어/일본어/영어 표시 언어 전환을 유지한다.

## 일본어 전용 로직 목록

앱 소스가 추가되면 아래 항목은 중국어 호환 레이어와 UI 명칭을 분리한다.

| 기존 일본어 개념 | 중국어 1차 대응 |
| --- | --- |
| `furigana` | `pinyin`과 같은 값으로 채우는 호환 필드 |
| `jlpt` | `HSK4`로 채우는 호환 필드 |
| `opic` | `band` 또는 난이도 그룹으로 채우는 호환 필드 |
| 일본어 한자/한국어 유사어 판단 | 중국어 단어장에서는 삭제 |

## 데이터 필드 매핑표

| 원본 필드 | 앱 표준 필드 | 비고 |
| --- | --- | --- |
| `word` | `word` | 중국어 단어 |
| `pinyin` | `pinyin`, `reading`, `furigana` | `furigana`는 호환 필드 |
| `meaning` | fallback meaning | 번역 누락 시 사용 |
| `translations.ko` | `meaning`, `translations.ko` | 기본 표시 언어 |
| `translations.en` | `translations.en` | 영어 표시 |
| `translations.ja` | `translations.ja` | 일본어 표시 |
| `partOfSpeech` | `pos` | 누락 1건은 변환 시 빈 문자열 또는 `unknown` 처리 필요 |
| `lessonId` | `lessonId` | 원본 레슨 유지 |
| `lessonId` | `step`, `level` | `Math.ceil(lessonId / 2)` |
| 없음 | `id` | 안정적 생성 필요 |
| 없음 | `hsk`, `jlpt` | `HSK4` 고정 |
| 없음 | `band`, `opic` | 1차 호환 필드 |
| 없음 | `example` | 1차는 빈 배열 |
